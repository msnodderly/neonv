# NeoNV v1.1.2

This patch release improves data safety and reliability while preserving existing note behavior.

### Fixes

- **Quit protection follows autosave correctly** — pressing Command-Q before autosave completes shows a confirmation instead of silently discarding the latest edit; once autosave succeeds, quitting proceeds normally without a false warning.
- **Rapid note switching stays in sync** — asynchronous file loads can no longer replace the editor with content from a previously selected note.
- **Org titles stay readable after external edits** — file-watcher refreshes continue to use `#+TITLE:` metadata instead of falling back to the raw first line.
- **Unicode-safe search highlighting** — notes containing characters whose case conversion changes length, such as Turkish `İ`, no longer crash or display misplaced highlights.
- **Consistent extension handling** — note discovery, renaming, and wiki-link resolution now share the same `.md`, `.txt`, and `.org` extension rules.

### Installation

Install or upgrade with Homebrew:

```bash
brew upgrade --cask msnodderly/tap/neonv
```

Alternatively, download the DMG and drag NeoNV to Applications.

**First launch for a manually downloaded unsigned build:**
1. Double-click NeoNV — macOS blocks it
2. Open **System Settings → Privacy & Security**
3. Click **Open Anyway** in the Security section
4. Enter your password when prompted
