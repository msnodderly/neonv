## NeoNV v1.1.1

A focused fix to the search field's Return behavior, restoring predictable note creation.

### Fixes

- **Return creates the title you typed** — typing a partial query such as `proj` and pressing Return now creates `proj.md`, even when notes matching `proj` are visible in the list. Return only opens an existing note when your text exactly identifies one (by normalized title, filename, basename, relative path, or sanitized destination); otherwise it creates. This restores the Notational Velocity-style search-or-create workflow that a previous change had regressed.
- **Explicit result navigation still opens matches** — press Down or Tab to move into the results list, then Return, to open a partial match.
- **Clearer submit hint** — the search field now states whether Return will create or open, alongside the match count.
- **No more silent Return** — a query that reduces to no usable filename (for example `.`, `..`, or `/`) no longer shows a misleading "create" hint and then do nothing.

### Installation

Download the DMG and drag NeoNV to Applications.

**First launch (unsigned app):**
1. Double-click NeoNV — macOS blocks with "cannot be opened"
2. Open **System Settings → Privacy & Security**
3. Click **"Open Anyway"** in Security section
4. Enter password when prompted
