## NeoNV v1.1.0

Smarter search (multi-word, full-content), a working command-line folder argument, and a batch of UI polish.

### New Features

- **Open a folder from the command line** — `open -na NeoNV --args ~/Documents/notes` (or run the binary directly with a path) opens that folder and saves it for future launches. The app sandbox that silently blocked this has been removed; your saved folder and settings migrate automatically on first launch.
- **Multi-word search matches all terms** — "company research" now finds notes containing both words anywhere, not just the exact phrase.
- **Search the whole note, not just the first 2 KB** — matches deep in a file now appear in results, and the row preview recenters on the match (with an "…" prefix and highlighting) so you can see why each note matched.
- **Toggle the file list with ⌘⇧B** — also listed in the keyboard shortcuts reference (⌘K).

### Improvements

- **File list defaults to 25% of the window width** — and the divider remains freely draggable.
- **Clearing a search reveals your note** — when you press Escape, the list scrolls the restored selection back into view.
- **Closing the window quits the app** — single-window behavior; the app can no longer relaunch into an invisible, windowless state.

### Fixes

- **Empty list on first open of a new folder** — opening a folder the app had never indexed could show "No notes found" even though notes were discovered; the list now populates reliably.
- **Symlinked folder paths** — notes folders reached through symlinks (e.g. under `/tmp`) no longer show mangled file paths in the list.
- **Stable list order** — notes sharing a modification date now keep a consistent order between launches.

### Installation

Download the DMG and drag NeoNV to Applications.

**First launch (unsigned app):**
1. Double-click NeoNV — macOS blocks with "cannot be opened"
2. Open **System Settings → Privacy & Security**
3. Click **"Open Anyway"** in Security section
4. Enter password when prompted
